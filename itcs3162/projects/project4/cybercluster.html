What is Clustering and How Does It Work?

Clustering is an unsupervised machine learning technique that groups data points based on inherent similarities without using pre-assigned labels. In this project, we will explore two clustering methods:

K-Means Clustering:

  • Process: The algorithm partitions the data into K clusters by iteratively assigning data points to the nearest centroid and recalculating the centroids.
  • Objective: It minimizes the within-cluster sum of squared distances (inertia).

Agglomerative (Hierarchical) Clustering:

  • Process: This method starts by treating every data point as an individual cluster and then gradually merges the closest clusters based on a chosen distance metric (e.g., single, complete, or average linkage) until a specified number of clusters remains.
  • Outcome: It produces a dendrogram, a hierarchy of clusters, that can be "cut" to yield clusters at different levels of granularity.

These approaches allow us to identify natural groupings in the network data, potentially flagging abnormal or suspicious sessions. en patterns.

Introduce the Data

The dataset used in this project is titled "cybersecurity_intrusion_data.csv" and was obtained from Kaggle. It contains 9,537 network session records with 11 features capturing various attributes of session activity. The ultimate goal is to detect anomalies or potential attacks using clustering.

Key Features:

  • session_id: Unique identifier for each session.
  • network_packet_size: Size (in bytes) of packets transferred during the session.
  • protocol_type: Network protocol used (e.g., TCP, UDP, ICMP).
  • login_attempts: Number of login attempts during the session.
  • session_duration: Duration of the session in seconds.
  • encryption_used: Indicates if encryption was employed during the session (contains missing values).
  • ip_reputation_score: Reputation score of the IP address (ranging from 0 to 1).
  • failed_logins: Number of failed login attempts.
  • browser_type: Type of browser used during the session.
  • unusual_time_access: Binary flag indicating if the session occurred at an unusual time.
  • attack_detected: Binary label indicating whether the session was identified as part of an attack.

Summary Statistics:

  • Records: 9,537
  • Features: 11
  • Missing Values:
    • encryption_used has 1,966 missing entries; all other features are complete.

Descriptive statistics reveal that, for example, the average network packet size is approximately 500 bytes, and the average session duration is about 792 seconds. This data provides a robust foundation for unsupervised learning, especially since we are not relying solely on the provided attack labels during training.

Pre-processing the Data

Effective clustering requires careful pre-processing. Our steps include:

  1. Encoding Categorical Variables:
    We will use Label Encoding to transform text-based features such as protocol_type, browser_type, and encryption_used (if treated categorically) into numeric values.

  2. Scaling Numerical Features:
    Distance-based algorithms like K-Means require features on a similar scale. Hence, we'll standardize numeric features (e.g., network_packet_size, session_duration, login_attempts, etc.) using StandardScaler.

  3. DataFrame for Clustering:
    We will combine the processed numeric and categorical features into a single DataFrame that serves as the input for our clustering algorithms.

Note: We will handle missing values in encryption_used appropriately (e.g., leave them as NaN for now or impute if necessary) based on our exploratory analysis.

Modeling (Clustering)

We will now apply two clustering algorithms to our pre-processed data:

  1. K-Means Clustering:

    • We will determine the optimal number of clusters using the Elbow Method and silhouette scores.
    • K-Means is efficient for large, scaled datasets and works well with spherical clusters.

  2. Agglomerative Clustering:

    • This hierarchical method offers an alternative view of the datas structure.
    • Using the 'ward' linkage minimizes within-cluster variance.

Both methods will help us explore how well we can segregate normal network sessions from potential attack sessions.

In[22]:
# Prepare the feature matrix for clustering
X = df_clustering.values

# Elbow Method: Compute inertia for a range of cluster numbers using K-Means
inertia_values = []
K_range = range(2, 10)
for k in K_range:
    km = KMeans(n_clusters=k, random_state=42)
    km.fit(X)
    inertia_values.append(km.inertia_)

plt.figure(figsize=(6, 4))
plt.plot(list(K_range), inertia_values, marker='o')
plt.title("Elbow Method: Optimal Number of Clusters")
plt.xlabel("Number of Clusters (k)")
plt.ylabel("Inertia")
plt.show()

# Silhouette Scores for each value of k
for k in K_range:
    km = KMeans(n_clusters=k, random_state=42)
    labels = km.fit_predict(X)
    score = silhouette_score(X, labels)
    print(f"k = {k}, Silhouette Score = {score:.4f}")

Impact Section

Potential Benefits:

  • Automated Anomaly Detection:
    Clustering helps highlight unusual network sessions that may indicate emerging cyber threats, reducing the burden of manual monitoring.
  • Operational Efficiency:
    By grouping similar network behaviors, system administrators can focus on investigating clusters of anomalous activity.

Potential Challenges and Ethical Considerations:

  • False Positives/Negatives:
    Incorrect cluster assignments may lead to legitimate sessions being flagged as attacks (and vice versa), affecting operational decisions.
  • Dataset Limitations:
    The dataset's age and composition may not fully capture modern attack techniques.
  • Privacy Concerns:
    Handling and analyzing network data, especially in real-world applications, requires attention to data privacy and protection measures.

Overall, while the clustering approach shows promise, further refinements and complementary techniques would be necessary for deployment in operational settings.
