Microsoft Teams UI Redesign (InnovateSync)

A capstone project focused on rethinking Microsoft Teams with cleaner navigation, Live Rooms, and a more focused collaboration experience – built as a production-quality prototype for my portfolio.

1. Project Overview Project Summary & Key Links

This capstone project reimagines Microsoft Teams as InnovateSync, a more focused and streamlined collaboration platform. The redesign introduces Live Rooms, a cleaner information architecture, and an updated visual style aimed at reducing clutter and improving productivity for students and professionals working in teams.

2. Your Contributions Features I Built & Technologies Used

My primary responsibility was taking the team’s UX direction and turning it into a working interface. I focused on front-end implementation, layout, and interaction behavior while keeping the design aligned with our Figma prototypes.

Features I Implemented

  • Live Rooms navigation layout – Added a dedicated Live Rooms entry in the left nav rail and designed the layout for joining and managing live collaboration spaces.
  • Top navigation bar – Implemented a custom header with the product logo, search, and user profile/avatar controls consistent across pages.
  • Teams, Chat, and Calendar pages – Built the main page layouts using semantic HTML and modular CSS so components can be reused across the app.
  • Responsive behavior – Ensured the core layouts adjust gracefully on smaller screens and laptop resolutions so the experience remains usable in different contexts.
  • JSON-driven content structure (planned) – Designed the structure for using JSON to populate Live Rooms and Teams data, making it easier to scale the prototype in future sprints.

Technologies Used

HTML5 CSS3 (custom layout & styling) Flexbox / CSS Grid Vanilla JavaScript JSON (data structure planning) Responsive & Accessible Design Practices

3. Your Journey What I Learned, Challenges, and Skills Gained

What I Learned

This project pushed me to think beyond just “making a page look good.” I had to consider user flows, navigation friction, and how people actually move between Teams, Chat, Live Rooms, and Calendar in a real workday. I also learned how to translate UX discussions and sketches into precise HTML/CSS structures that other developers could understand and extend.

Challenges I Overcame

The biggest challenges were balancing scope with quality and keeping the layouts consistent across multiple pages. As the design evolved, I had to refactor CSS to avoid duplication and ensure components were reusable. Coordinating changes with my teammates while we all iterated on features also forced me to communicate more clearly and document decisions.

Skills I Gained

  • Improved at structuring large front-end codebases with reusable components.
  • Gained experience aligning UI implementation with Figma prototypes and UX goals.
  • Strengthened communication skills when merging design, development, and feedback.
  • Learned how to design navigation and layouts that scale across multiple feature areas.
